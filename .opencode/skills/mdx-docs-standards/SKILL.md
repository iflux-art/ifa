# MDX 文档创建标准

## 概述

本文档定义了创建 MDX 文档时必须遵循的标准，适用于 `src/content/` 目录下的所有文档。

## Frontmatter 标准

### 必须字段

```yaml
---
title: 文档标题
description: 文档描述
category: 分类名
tags:
  - 标签1
  - 标签2
  - 标签3
date: 2026-02-22
---
```

### 重要规则

- **tags 字段只能有 3 个标签**，选择最符合文档内容的 3 个标签
- tags 应按相关性排序，最相关的放在第一位

## 标题标准

### 禁止重复

- **禁止使用重复的二级标题（##）**
- **禁止使用重复的三级标题（###）**
- 每个标题在文档中必须唯一

## 分隔线标准

### 禁止使用

- **二级标题（##）之间禁止使用分隔线（---）**
- 只允许在 frontmatter 中使用 `---`

## 创建文档流程

1. 确定文档主题和目标
2. 搜索相关资料获取内容
3. 编写 frontmatter（tags 只用 3 个）
4. 编写文档内容，避免重复标题
5. 不在二级标题间加分隔线
6. 检查无报错后完成

## 检查清单

- [ ] frontmatter 中 tags 只有 3 个
- [ ] 没有重复的二级标题
- [ ] 没有重复的三级标题
- [ ] 二级标题之间没有分隔线
- [ ] 文档内容完整准确
